# Plan 11-01: Backend — Multi-Account Session Migration & Account Management

## Goal
Migrate session cookies from `users` table to `steam_accounts` table so each linked Steam account has its own session. Add account CRUD endpoints and premium gate.

## Context
- `steam_accounts` table already exists with `user_id`, `steam_id`, `display_name`, `avatar_url`, `trade_token`
- `inventory_items` already references `steam_account_id` (not user_id)
- Session columns (`steam_session_id`, `steam_login_secure`, `steam_access_token`, `steam_refresh_token`, `session_method`, `session_updated_at`, `wallet_currency`) are currently on `users` table
- All session service methods take `userId` — must change to `accountId`
- Sell operations use `ensureValidSession(userId)` — must specify which account

## Tasks

### 1. Database Migration — Move Session Columns to steam_accounts

File: `backend/src/db/migrations/` (new file: `009_multi_account_sessions.sql`)

```sql
-- Add session columns to steam_accounts
ALTER TABLE steam_accounts ADD COLUMN steam_session_id TEXT;
ALTER TABLE steam_accounts ADD COLUMN steam_login_secure TEXT;
ALTER TABLE steam_accounts ADD COLUMN steam_access_token TEXT;
ALTER TABLE steam_accounts ADD COLUMN steam_refresh_token TEXT;
ALTER TABLE steam_accounts ADD COLUMN session_method VARCHAR(20);
ALTER TABLE steam_accounts ADD COLUMN session_updated_at TIMESTAMPTZ;
ALTER TABLE steam_accounts ADD COLUMN wallet_currency INTEGER;

-- Add active_account_id to users for "currently selected" account
ALTER TABLE users ADD COLUMN active_account_id INTEGER REFERENCES steam_accounts(id);

-- Migrate existing session data from users to their primary steam_account
UPDATE steam_accounts sa
SET steam_session_id = u.steam_session_id,
    steam_login_secure = u.steam_login_secure,
    steam_access_token = u.steam_access_token,
    steam_refresh_token = u.steam_refresh_token,
    session_method = u.session_method,
    session_updated_at = u.session_updated_at,
    wallet_currency = u.wallet_currency
FROM users u
WHERE sa.user_id = u.id AND sa.steam_id = u.steam_id;

-- Set active_account_id to primary account
UPDATE users u
SET active_account_id = sa.id
FROM steam_accounts sa
WHERE sa.user_id = u.id AND sa.steam_id = u.steam_id;

-- Drop session columns from users (after migration verified)
ALTER TABLE users DROP COLUMN IF EXISTS steam_session_id;
ALTER TABLE users DROP COLUMN IF EXISTS steam_login_secure;
ALTER TABLE users DROP COLUMN IF EXISTS steam_access_token;
ALTER TABLE users DROP COLUMN IF EXISTS steam_refresh_token;
ALTER TABLE users DROP COLUMN IF EXISTS session_method;
ALTER TABLE users DROP COLUMN IF EXISTS session_updated_at;
ALTER TABLE users DROP COLUMN IF EXISTS wallet_currency;

-- Add steam_account_id to sell_operations for per-account tracking
ALTER TABLE sell_operations ADD COLUMN steam_account_id INTEGER REFERENCES steam_accounts(id);
-- Backfill from primary account
UPDATE sell_operations so
SET steam_account_id = sa.id
FROM users u
JOIN steam_accounts sa ON sa.user_id = u.id AND sa.steam_id = u.steam_id
WHERE so.user_id = u.id;
```

Also update `migrate.ts` to run this migration file.

### 2. Refactor SteamSessionService — accountId Instead of userId

File: `backend/src/services/steamSession.ts`

Change ALL methods to accept `accountId: number` (steam_accounts.id) instead of `userId`:

- `getSession(accountId)` — `SELECT ... FROM steam_accounts WHERE id = $1`
- `saveSession(accountId, session)` — `UPDATE steam_accounts SET ... WHERE id = $1`
- `startQRSession(accountId)` — pass accountId through pending session
- `pollQRSession(nonce, accountId)` — save to correct account
- `startCredentialLogin(accountId, username, password)` — same
- `submitGuardCode(nonce, code)` — save to pending session's accountId
- `handleClientToken(accountId, tokenData)` — save to correct account
- `getSessionStatus(accountId)` — check specific account
- `refreshSession(accountId)` — refresh specific account
- `ensureValidSession(accountId)` — validate specific account

Add helper: `getActiveAccountId(userId: number): Promise<number>` — reads `users.active_account_id`, falls back to first linked account.

Update wallet currency detection to write to `steam_accounts` instead of `users`.

### 3. Account Management Endpoints

File: `backend/src/routes/auth.ts` (add new routes)

```

POST /api/accounts/link         — Link new Steam account (starts OpenID flow for additional account)
DELETE /api/accounts/:accountId — Unlink account (delete steam_account + its inventory)
PUT  /api/accounts/:accountId/active — Set as active account
GET  /api/accounts/:accountId/session/status — Session status for specific account
```

**GET /api/accounts** response:

**POST /api/accounts/link** — Premium gate check:
```typescript
// Check premium status
if (!req.isPremium) {
  const { rows } = await pool.query(
    'SELECT COUNT(*) FROM steam_accounts WHERE user_id = $1', [req.userId]
  );
  if (parseInt(rows[0].count) >= 1) {
    return res.status(403).json({ error: 'premium_required', message: 'Upgrade to link multiple accounts' });

// Start OpenID flow for additional account (similar to initial auth but inserts into steam_accounts only)
```

**DELETE /api/accounts/:accountId** — Cascade cleanup:
- Cannot delete last/primary account
- Delete inventory_items, sell_operations, alerts for that account
- If deleted account was active, switch active to primary

### 4. Update Session Routes — Account-Scoped

File: `backend/src/routes/session.ts`

All session routes need `accountId` parameter. Two approaches:
- **Option A**: Add `?accountId=X` query param (default to active account)
- **Option B**: Use `active_account_id` from users table always

Go with **Option A** for flexibility, falling back to active account:

```typescript
// Helper at top of session routes
function getAccountId(req: AuthRequest): number {
  return parseInt(req.query.accountId as string) || null;
}

// In each route handler:
const accountId = getAccountId(req) || await SteamSessionService.getActiveAccountId(req.userId!);
```

Update all session route handlers:
- `POST /session/qr/start` → `startQRSession(accountId)`
- `GET /session/qr/poll/:nonce` → `pollQRSession(nonce, accountId)`
- `POST /session/login` → `startCredentialLogin(accountId, ...)`
- `POST /session/guard` → stays nonce-based (accountId from pending session)
- `POST /session/token` → `handleClientToken(accountId, ...)`
- `POST /session/refresh` → `refreshSession(accountId)`
- `GET /session/status` → `getSessionStatus(accountId)`
