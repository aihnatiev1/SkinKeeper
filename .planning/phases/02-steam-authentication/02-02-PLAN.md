  <files>lib/features/auth/session_provider.dart, lib/features/auth/steam_session_screen.dart, lib/features/auth/widgets/qr_auth_tab.dart, lib/features/auth/widgets/credentials_auth_tab.dart, lib/features/auth/widgets/clienttoken_auth_tab.dart, lib/features/auth/widgets/session_status_widget.dart</files>
  <action>
1. Create `lib/features/auth/session_provider.dart`:

   a. `sessionStatusProvider` - an `AsyncNotifierProvider<SessionStatusNotifier, String>`:
   - `build()` calls `GET /api/session/status`, returns status string ('valid'/'expiring'/'expired'/'none')
   - `refresh()` method to re-fetch status
   - Use ApiClient from apiClientProvider

   b. `qrAuthProvider` - a `StateNotifierProvider` or `AsyncNotifierProvider` managing QR auth state:
   - State: `{ qrImage: String?, nonce: String?, status: String, loading: bool, error: String? }`
   - `startQR()` method: POST /api/session/qr/start, stores qrImage and nonce
   - `pollQR()` method: GET /api/session/qr/poll/:nonce, returns status
   - Use a simple data class for QR auth state

   c. `credentialAuthProvider` - managing login+guard state:
   - State: `{ nonce: String?, guardRequired: bool, status: String, loading: bool, error: String? }`
   - `login(username, password)`: POST /api/session/login
   - `submitGuard(code)`: POST /api/session/guard

   d. `clientTokenAuthProvider` - managing token submission:
   - `submitToken(steamLoginSecure)`: POST /api/session/token

2. Create `lib/features/auth/widgets/session_status_widget.dart`:
   - A ConsumerWidget that watches sessionStatusProvider
   - Displays icon + label based on status:
     - 'valid': green check_circle, "Session Active"
     - 'expiring': orange warning, "Expiring Soon"
     - 'expired': red error, "Session Expired"
     - 'none': grey help_outline, "Not Connected"
   - On tap: navigate to steam_session_screen
   - Use compact Row layout (icon + text) suitable for app bar or settings

3. Create `lib/features/auth/widgets/qr_auth_tab.dart`:
   - ConsumerStatefulWidget
   - On init: call startQR() from provider
   - Display QR image using Image.memory (decode base64 from data URI, strip "data:image/png;base64," prefix)
   - Show "Scan with Steam Mobile App" instruction text
   - Poll every 3 seconds using a Timer. On 'authenticated': show success, refresh sessionStatusProvider, pop back. On 'expired': show "QR expired" with a "Refresh" button that calls startQR() again.
   - Cancel timer on dispose

4. Create `lib/features/auth/widgets/credentials_auth_tab.dart`:
   - ConsumerStatefulWidget with TextEditingControllers for username, password, guardCode
   - Step 1: Username + password fields with "Login" button -> calls login()
   - Step 2 (if guardRequired): Shows guard code field with "Submit Code" button -> calls submitGuard()
   - On success: show success message, refresh sessionStatusProvider, pop back
   - On error: show error in SnackBar
   - Password field uses obscureText: true

5. Create `lib/features/auth/widgets/clienttoken_auth_tab.dart`:
   - ConsumerStatefulWidget with TextEditingController for token paste
   - Instructions text: "1. Open Steam in browser, 2. Navigate to steamcommunity.com/chat/clientjstoken, 3. Copy the steamLoginSecure value, 4. Paste below"
   - TextField for steamLoginSecure value
   - "Submit" button -> calls submitToken()
   - On success: refresh sessionStatusProvider, pop back

6. Create `lib/features/auth/steam_session_screen.dart`:
   - Scaffold with AppBar titled "Steam Session"
   - DefaultTabController with 3 tabs: "QR Code", "Login", "Token"
   - TabBarView containing QrAuthTab, CredentialsAuthTab, ClientTokenAuthTab
   - Show current SessionStatusWidget at top of screen (above tabs)

   Follow existing app style: dark theme, consistent with login_screen.dart patterns.
