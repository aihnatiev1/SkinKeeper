**backend/src/infra/SteamGateway.ts** — Wraps `steamRequest` with three layers:

1. **Token bucket rate limiter:**
   - Configurable via `STEAM_RATE_LIMIT` env var (default 10 req/s).
   - Internal: track `tokens` (starts at limit), `lastRefill` timestamp. On each request: refill tokens based on elapsed time, if tokens > 0 decrement and proceed, else wait until a token is available (`sleep(1000/limit)`).
   - This is a global limiter for ALL Steam calls going through the gateway.

2. **Request dedup:**
   - `Map<string, Promise<SteamResponse>>` keyed by `method + url + sorted_params_hash`.
   - If key exists and entry is < 5s old, return the existing promise.
   - On resolve/reject, schedule removal after 5s (not immediately — let concurrent callers benefit).
   - IMPORTANT: Only dedup GET requests. POST requests always execute.

3. **Circuit breaker:**
   - Track `consecutiveFailures` per domain (steamcommunity.com, api.steampowered.com).
   - If failures > threshold (env `STEAM_CIRCUIT_THRESHOLD`, default 5), open circuit for `STEAM_CIRCUIT_COOLDOWN_MS` (default 60000ms).
   - While open, reject immediately with `CircuitOpenError`.
   - On success, reset counter.
   - `getCircuitState()` returns `{ domain, state: 'closed'|'open', failures, openUntil }`.

4. **Metrics:**
   - Wrap existing `getSteamClientMetrics()` and add: dedupHits, circuitBreaks, rateLimitWaits.
   - `getGatewayMetrics()` export.

Main export:
```typescript
export const SteamGateway = {
  async request<T>(opts: SteamRequestOptions): Promise<SteamResponse<T>>,
  getMetrics(): GatewayMetrics,
  getCircuitState(): CircuitState[],
};
```

The `request` method flow: check circuit -> dedup check -> rate limit wait -> steamRequest(opts) -> record metrics -> update circuit state -> return.

**backend/src/infra/ResponseCache.ts** — API response cache:

Uses `TTLCache` internally. Provides:
```typescript
export class ResponseCache {
  constructor()
  get<T>(key: string): T | undefined
  set<T>(key: string, data: T, ttlMs: number): void
  invalidate(pattern: string): number  // delete keys matching pattern, return count
  getStats(): { size, hits, misses, hitRate }
}
```

Default TTLs (configurable via env):
- `CACHE_TTL_PORTFOLIO`: 60000 (1min)
- `CACHE_TTL_INVENTORY`: 30000 (30s)
- `CACHE_TTL_PRICES`: 120000 (2min)

Module-level singleton: `export const responseCache = new ResponseCache()`.

Register with cacheRegistry for admin monitoring: `registerCache('responseCache', ...)`.

The `invalidate(pattern)` method iterates keys and deletes those matching the pattern (simple string includes, not regex). Used for cache-busting on writes (e.g., after inventory refresh, invalidate all keys containing the userId).

2. Add slow query logging by monkey-patching pool.query:
   ```typescript
   const originalQuery = pool.query.bind(pool);
   const SLOW_QUERY_MS = parseInt(process.env.PG_SLOW_QUERY_MS || '500');

   pool.query = async function(...args: any[]) {
     const start = Date.now();
     try {
       const result = await originalQuery(...args);
       const duration = Date.now() - start;
       if (duration > SLOW_QUERY_MS) {
         const queryText = typeof args[0] === 'string' ? args[0].substring(0, 200) : '(QueryConfig)';
         console.warn(`[Pool] SLOW QUERY (${duration}ms): ${queryText}`);
       }
       return result;
     } catch (err) {
       const duration = Date.now() - start;
       if (duration > SLOW_QUERY_MS) {
         console.warn(`[Pool] SLOW QUERY FAILED (${duration}ms)`);
       }
       throw err;
     }
   } as typeof pool.query;
   ```

   ```typescript
       totalCount: pool.totalCount,
       idleCount: pool.idleCount,
       waitingCount: pool.waitingCount,
       max: pool.options.max,
     };
   }
   ```
