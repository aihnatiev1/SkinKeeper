**backend/src/infra/types.ts** — Shared queue types:
```typescript
export type JobStatus = 'waiting' | 'active' | 'completed' | 'failed';

export interface Job<T = unknown> {
  id: string;
  name: string;
  data: T;
  status: JobStatus;
  progress: number;        // 0-100 or item count
  result?: unknown;
  error?: string;
  createdAt: number;       // Date.now()
  startedAt?: number;
  completedAt?: number;
}

export interface QueueOptions {
  concurrency: number;     // max parallel jobs
  jobTtlMs: number;        // auto-cleanup completed jobs (default 30min)
}

export type JobHandler<T = unknown> = (
  job: Job<T>,
  updateProgress: (progress: number) => void
) => Promise<unknown>;
```

**backend/src/infra/JobQueue.ts** — In-memory queue implementation:

Class `JobQueue<T>`:
- Constructor takes `name: string` and `options: Partial<QueueOptions>`. Defaults: concurrency from env `QUEUE_CONCURRENCY` or 3, jobTtlMs 30min.
- `add(name: string, data: T): Job<T>` — creates job with UUID id, status 'waiting', pushes to internal queue, calls `_process()`.
- `process(handler: JobHandler<T>): void` — registers the handler (one per queue).
- `getJob(id: string): Job<T> | undefined` — returns job by id.
- `getStats(): { name: string; waiting: number; active: number; completed: number; failed: number; concurrency: number }`.
- Private `_process()`: while active < concurrency and waiting jobs exist, dequeue next, set status 'active', call handler with progress callback, set 'completed'/'failed' on resolve/reject.
- Private cleanup interval (every 5min, unref'd): remove completed/failed jobs older than jobTtlMs.

Module-level queue registry:
- `const queues = new Map<string, JobQueue<any>>()`
- `getQueue<T>(name: string, options?: Partial<QueueOptions>): JobQueue<T>` — get-or-create pattern.
- `getAllQueueStats(): Record<string, QueueStats>` — returns stats for all registered queues.

Important: The interface mirrors BullMQ's `Queue.add()`, `Worker.process()`, `Job.progress` pattern. When migrating to Redis, only the constructor changes — callers stay the same.

Env var: `QUEUE_CONCURRENCY` (default 3). Per-queue override: `QUEUE_CONCURRENCY_INVENTORY=5`, `QUEUE_CONCURRENCY_INSPECT=2`.
