# Plan 13-02: Flutter — Widget Data Provider & Background Refresh

## Goal
Push portfolio data to the widget whenever it changes, schedule background refresh, and handle deep links from widget taps.

## Context
- WidgetService (13-01) provides `updateWidget()` to push data to native widgets
- Portfolio provider already fetches totalValue, change24h, change24hPct, itemCount
- P/L provider already fetches totalProfit for premium users
- CacheService (Phase 12) has cached portfolio data
- Need: update widget after every portfolio fetch, schedule background task, handle widget tap deep link

## Tasks

### 2. Background Widget Refresh

File: `lib/core/widget_service.dart`

Add background fetch registration using `home_widget`:

```dart
static Future<void> registerBackgroundCallback() async {
  await HomeWidget.registerInteractivityCallback(backgroundCallback);
}

@pragma('vm:entry-point')
static Future<void> backgroundCallback(Uri? uri) async {
  // This is called when the widget requests an update
  // Re-read from CacheService and push to widget
  await CacheService.init();
  final portfolio = CacheService.getPortfolio();
  if (portfolio != null) {
    await updateWidget(
      totalValue: portfolio['totalValue'] ?? '--',
      change24h: portfolio['change24h'] ?? '--',
      change24hPct: portfolio['change24hPct'] ?? '--',
      isPositive: portfolio['isPositive'] ?? true,
      itemCount: portfolio['itemCount'] ?? 0,
    );
  }
}
```

### 4. Update Widget on App Foreground

File: `lib/main.dart`

When app comes to foreground, push latest data to widget:

```dart
class _AppLifecycleObserver extends WidgetsBindingObserver {
  final WidgetRef ref;
  _AppLifecycleObserver(this.ref);

  @override
  void didChangeAppLifecycleState(AppLifecycleState state) {
    if (state == AppLifecycleState.resumed) {
      // Refresh widget with latest cached data
      final portfolio = CacheService.getPortfolio();
      if (portfolio != null) {
        WidgetService.updateWidget(...);
      }
    }
  }
}
```
