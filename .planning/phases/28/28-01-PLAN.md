Create `lib/features/auth/widgets/session_gate_screen.dart` -- a full-screen ConsumerStatefulWidget with WidgetsBindingObserver (for app lifecycle clipboard detection).

**Structure:**

The screen has two visual states:
- State A: Gate content (default) -- shows value copy + connect flow
- State B: ConnectProgressOverlay (after successful auth) -- then pops with true

**Gate content layout (State A), scrollable SingleChildScrollView:**

1. **AppBar**: Back button (pops with false), title "Enable Full Access"

2. **Value proposition section** (top, inside a container with primary-tinted background):
   - Subtitle: "Steam requires this extra step to protect your items"
   - Feature chips row (Wrap widget): "Sell Items", "Trade", "Market History", "Profit & Loss"
   - Use same `_featureChip` pattern from ClientTokenAuthTab but in a Wrap for better layout

3. **Step cards section** -- Reuse the `_StepCard` widget pattern from ClientTokenAuthTab:
   - Track `_step` (1 or 2) and `_waitingForReturn` state
   - Step 1: "Sign in to Steam" / "We'll open Steam in your browser. Log in if needed." / Button: "Open Steam" with Icons.open_in_new_rounded
   - Step 2: "Copy your session" / "We'll open a special page. Tap Select All then Copy." / Button: "Open & Copy" with Icons.content_copy_rounded
   - Copy the `_StepCard` class from clienttoken_auth_tab.dart (it's private, so duplicate it)

4. **Auto-status banner** -- Same pattern as ClientTokenAuthTab._buildStatusBanner():
   - 'detecting': "Reading your clipboard..."
   - 'found': "Session found! Connecting..."
   - 'not_logged_in': "Not signed in to Steam yet" + retry button
   - 'not_found': "Couldn't detect automatically"

5. **Manual paste section** (shown when autoStatus is 'not_found' or user taps "Paste manually"):
   - TextField + Connect button (same as ClientTokenAuthTab)

6. **"Having trouble?" collapsible section** (ExpansionTile or custom toggle):
   - Section title: "Have another device?" with expand icon
   - When expanded: embed QR auth flow inline
   - Create a `_QrFallbackSection` that:
     - Calls `ref.read(qrAuthProvider.notifier).startQR()` when expanded
     - Shows QR image from qrAuthProvider.qrImage (base64 decode + Image.memory)
     - Polls every 5s via Timer.periodic calling pollQR()
     - Shows "Waiting for confirmation..." shimmer while polling
     - On 'authenticated': trigger success flow
     - On 'expired': show refresh button
     - Cancel timer on collapse or dispose
   - Below QR: small text "Open Steam Guard on another device and scan this code"

**Lifecycle handling (WidgetsBindingObserver):**
- `didChangeAppLifecycleState(resumed)`: if `_waitingForReturn` is true:
  - If `_step == 1`: advance to step 2
  - If `_step == 2`: call `_tryAutoFillFromClipboard()` (same logic as ClientTokenAuthTab)

**Token extraction and submission:**
- Copy `_extractToken()` and `_isNotLoggedInJson()` helper methods from ClientTokenAuthTab
- On successful token extraction: call `ref.read(clientTokenAuthProvider.notifier).submitToken(token)`
- Listen to `clientTokenAuthProvider` for status changes:
  - 'authenticated': refresh sessionStatusProvider, then switch to ConnectProgressOverlay state
  - 'error': show error snackbar

**Success flow:**
- When clientTokenAuthProvider or qrAuthProvider reports 'authenticated':
  1. Call `ref.read(sessionStatusProvider.notifier).refresh()`
  2. Set state to show ConnectProgressOverlay
  3. ConnectProgressOverlay.onComplete -> `Navigator.of(context).pop(true)`

**Dismiss flow:**
- Back button or swipe back: `Navigator.of(context).pop(false)`

**Important implementation details:**
- Unfocus keyboard before popping (FocusManager.instance.primaryFocus?.unfocus()) to prevent EditableTextState crash
- Do NOT set sessionLinkModeProvider -- this is NOT account linking, it's session reconnect for existing active account
- Import clientTokenAuthProvider, qrAuthProvider, sessionStatusProvider from session_provider.dart
- Import ConnectProgressOverlay from connect_progress_overlay.dart
- Use dark theme styling consistent with AppTheme (bg, surface, border colors)
- The QR fallback section should dispose its poll timer in its own state management
