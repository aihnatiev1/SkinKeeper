# Plan 11-02: Flutter — Account Linking, Switching & Per-Account UI

## Goal
Add account management UI: link additional Steam accounts, switch active account, show account badges on inventory items, per-account session management. Premium gate for multiple accounts.

## Context
- Backend (11-01) provides: `GET /accounts`, `POST /accounts/link`, `DELETE /accounts/:id`, `PUT /accounts/:id/active`
- Session routes accept `?accountId=X` parameter
- Sell operations accept `accountId` in body
- Inventory returns `accountId` + `accountSteamId` per item
- Settings screen already has "Linked Accounts" button with empty `onTap`
- User model (`SteamUser`) only has single steamId
- Premium gate: free = 1 account, premium = unlimited

```dart
final accountsProvider = AsyncNotifierProvider<AccountsNotifier, List<SteamAccount>>(() => AccountsNotifier());

class AccountsNotifier extends AsyncNotifier<List<SteamAccount>> {
  @override
  Future<List<SteamAccount>> build() async {
    final response = await ref.read(apiClientProvider).get('/accounts');
    final data = response.data;
    return (data['accounts'] as List).map((a) => SteamAccount.fromJson(a)).toList();
  }

  Future<void> setActive(int accountId) async {
    await ref.read(apiClientProvider).put('/accounts/$accountId/active');
    ref.invalidateSelf();
    // Also invalidate inventory to reload with new active account context
    ref.invalidate(inventoryProvider);
  }

  Future<void> unlinkAccount(int accountId) async {
    await ref.read(apiClientProvider).delete('/accounts/$accountId');
    ref.invalidateSelf();
    ref.invalidate(inventoryProvider);
```
