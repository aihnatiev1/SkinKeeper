<task type="auto">
  <name>Task 2: Sell operations service with async batch processing</name>
  <files>backend/src/services/sellOperations.ts</files>
  <action>
Create `backend/src/services/sellOperations.ts`:

1. `createOperation(userId, items: {assetId, marketHashName, priceCents}[])`:
   - Insert into sell_operations (status: 'pending')
   - Insert all items into sell_operation_items (status: 'queued')
   - Return operation id
   - Start async processing via `processOperation(operationId)`

2. `processOperation(operationId)`:
   - Update operation status to 'in_progress'
   - Get user's session via SteamSessionService, validate + refresh if needed
   - Loop through queued items:
     - Update item status to 'listing'
     - Call `sellItem(session, assetId, priceCents)` from market.ts
     - On success: update to 'listed', set requires_confirmation flag, increment operation.succeeded
     - On failure: update to 'failed', set error_message, increment operation.failed
     - Increment daily sell volume
     - **Wait 3 seconds** between items (adaptive: if error, wait 5s, if two errors in a row, wait 10s)
     - If session becomes invalid mid-batch: attempt refresh, if fail abort remaining as 'failed'
   - After all items: update operation status to 'completed', set completed_at

3. `getOperation(operationId, userId)`:
   - Return operation with all items and their statuses
   - Include summary: total, succeeded, failed, pending count

4. `cancelOperation(operationId, userId)`:
   - Update operation status to 'cancelled'
   - Update all 'queued' items to 'cancelled' (don't cancel in-progress items, wait for them)

5. `getDailyVolume(userId)`:
   - Query sell_volume for today
   - Return { count, limit: 200, warningAt: 150 }

6. `incrementVolume(userId)`:
   - INSERT INTO sell_volume ... ON CONFLICT DO UPDATE SET count = count + 1

Import and use pool from db/pool.ts, SteamSessionService, and sellItem from market.ts.
  </action>

<task type="auto">
  <name>Task 3: Session auto-refresh endpoint</name>
  <files>backend/src/services/steamSession.ts, backend/src/routes/session.ts</files>
  <action>
1. In `steamSession.ts`, add method `refreshSession(userId)`:
   - Load user's encrypted `steam_refresh_token` from DB
   - If no refresh token: return { refreshed: false, reason: 'no_refresh_token' }
   - Attempt refresh using steam-session library (LoginSession.refreshAccessToken or equivalent)
   - If success: update `steam_login_secure`, `steam_session_id`, `steam_access_token`, `session_updated_at` in DB
   - Return { refreshed: true }
   - If refresh fails: return { refreshed: false, reason: 'refresh_failed' }

2. Add method `ensureValidSession(userId)`:
   - Call getSessionStatus(userId)
   - If 'valid': return session
   - If 'expiring': attempt refreshSession(userId), return session
   - If 'expired' or 'none': attempt refreshSession(userId), if fails throw SESSION_EXPIRED error
   - This is the method sell operations will call before starting

3. In `session.ts`, add endpoint:
   - `POST /api/session/refresh` (auth required):
     - Call refreshSession(req.userId)
     - Return { refreshed, status, reason? }
