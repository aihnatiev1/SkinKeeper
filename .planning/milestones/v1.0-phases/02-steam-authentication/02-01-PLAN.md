3. Extend SteamSessionService in steamSession.ts with these additions:

   a. Add imports at top:
   ```typescript
   import { LoginSession, EAuthTokenPlatformType } from 'steam-session';
   import QRCode from 'qrcode';
   import crypto from 'crypto';
   ```

   b. Add PendingSession interface and static map inside the class:
   ```typescript
   interface PendingSession {
     loginSession: LoginSession;
     createdAt: Date;
     userId: number;
     status: 'pending' | 'guard_required' | 'authenticated' | 'expired';
     cookies?: SteamSession;
     guardCallback?: (code: string) => void;
   }
   ```
   Store as `private static pendingSessions = new Map<string, PendingSession>();`

   c. Add static cleanup interval (5-minute TTL on pending sessions) using `setInterval` in a static initialization block or a `static { }` block.

   d. Add `static async startQRSession(userId: number)` method:
   - Create `new LoginSession(EAuthTokenPlatformType.WebBrowser)`
   - Call `session.startWithQR()`
   - Generate QR image via `QRCode.toDataURL(startResult.qrChallengeUrl)`
   - Generate nonce via `crypto.randomUUID()`
   - Listen for `authenticated` event: call `session.getWebCookies()`, parse cookies via `parseCookies()`, set status to 'authenticated'
   - Listen for `timeout` event: set status to 'expired'
   - Store in pendingSessions map
   - Return `{ qrImage, nonce }`

   e. Add `static async pollQRSession(nonce: string, userId: number)` method:
   - Look up pending session by nonce, verify userId matches
   - If status === 'authenticated' and cookies exist: call `saveSession(userId, cookies)`, save session_method='qr' and refresh token to DB, delete from map, return `{ status: 'authenticated' }`
   - If status === 'expired': delete from map, return `{ status: 'expired' }`
   - Otherwise return `{ status: 'pending' }`

   f. Add `static async startCredentialLogin(userId: number, username: string, password: string)` method:
   - Create LoginSession, set up `steamGuard` event handler that stores callback and sets status to 'guard_required'
   - Set up `authenticated` event handler same as QR flow
   - Call `session.startWithCredentials({ accountName: username, password })`
   - Return `{ nonce, guardRequired: pending.status === 'guard_required' }`

   g. Add `static async submitGuardCode(nonce: string, code: string)` method:
   - Look up pending session, verify guardCallback exists
   - Call guardCallback(code)
   - Wait up to 5 seconds (poll every 500ms) for status to become 'authenticated'
   - If authenticated: saveSession, cleanup, return session
   - If still pending after timeout: return null (code may be wrong, client can retry)

   h. Add `static async handleClientToken(userId: number, tokenData: { steamLoginSecure: string; steamId?: string })` method:
   - Extract sessionId via existing `extractSessionId(tokenData.steamLoginSecure)`
   - If sessionId is null, return null
   - Build SteamSession object, call saveSession, save session_method='clienttoken'
   - Return the session

   i. Add `static async getSessionStatus(userId: number)` method:
   - Query DB for steam_login_secure and session_updated_at
   - If no steam_login_secure: return 'none'
   - Check session_updated_at: if > 20 hours ago, return 'expiring'
   - Call validateSession() with decrypted session
   - Return 'valid' if passes, 'expired' if fails

   j. Add `private static parseCookies(cookieStrings: string[]): SteamSession` helper:
   - Parse cookie strings from getWebCookies() format
   - Extract sessionid and steamLoginSecure values
   - Return SteamSession object

   IMPORTANT: Do NOT store raw passwords. Username/password are used only in the startWithCredentials call and then discarded.
   IMPORTANT: The `steam-session` steamGuard event fires with `(domain, callback, lastCodeWrong)`. Store the callback in pendingSession.guardCallback.
  </action>

<task type="auto">
  <name>Task 2: Create session routes and mount in Express app</name>
  <files>backend/src/routes/session.ts, backend/src/index.ts</files>
  <action>
1. Create `backend/src/routes/session.ts` with Express Router:

   Import SteamSessionService, authMiddleware, AuthRequest.

   a. `POST /qr/start` (auth required):
   - Call `SteamSessionService.startQRSession(req.userId)`
   - Return `{ qrImage, nonce }` with 200
   - On error: 500 with `{ error: "Failed to start QR session" }`

   b. `GET /qr/poll/:nonce` (auth required):
   - Call `SteamSessionService.pollQRSession(req.params.nonce, req.userId)`
   - Return `{ status }` with 200

   c. `POST /login` (auth required):
   - Extract `{ username, password }` from req.body
   - Validate both fields present, return 400 if missing
   - Call `SteamSessionService.startCredentialLogin(req.userId, username, password)`
   - Return `{ nonce, guardRequired }` with 200
   - On error: return 401 with `{ error: "Login failed" }` (don't leak Steam error details)

   d. `POST /guard` (auth required):
   - Extract `{ nonce, code }` from req.body
   - Validate both present, return 400 if missing
   - Call `SteamSessionService.submitGuardCode(nonce, code)`
   - If result: return `{ status: "authenticated" }` with 200
   - If null: return `{ status: "pending", message: "Code may be incorrect, try again" }` with 200

   e. `POST /token` (auth required):
   - Extract `{ steamLoginSecure }` from req.body
   - Validate present, return 400 if missing
   - Call `SteamSessionService.handleClientToken(req.userId, { steamLoginSecure })`
   - If result: return `{ status: "authenticated" }` with 200
   - If null: return 400 with `{ error: "Could not extract session from token" }`

   f. `GET /status` (auth required):
   - Call `SteamSessionService.getSessionStatus(req.userId)`
   - Return `{ status }` with 200

   All routes use authMiddleware. All routes have try/catch returning 500 on unexpected errors.

2. Mount in `backend/src/index.ts`:
   - Import: `import sessionRouter from './routes/session.js';`
   - Add: `app.use('/api/session', sessionRouter);`
   - Place after existing route mounts
