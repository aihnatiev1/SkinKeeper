# Plan 14-01: Backend Refactoring — DB, Steam API, Scrapers, Performance

## Goal
Refactor the entire backend for production-grade performance, reliability, and maintainability. Fix all identified issues: connection pooling, unbounded caches, race conditions, input validation, error handling gaps, and TypeScript strictness.

## Context
Backend is Express.js 5 + TypeScript + PostgreSQL 17. 11 route files, 18 service files, ~3000 lines total. Current state is functional but has gaps: no pool config, unbounded Maps, `any` types, missing input validation, fire-and-forget async gaps, and fragile regex scrapers.

---

### Task 3: Fix Auth Middleware Race Condition
**Files:** `backend/src/middleware/auth.ts`

1. Convert `requirePremium` from `.then()/.catch()` to `async/await`:
   ```typescript
   async function requirePremium(req, res, next) {
     try {
       const result = await pool.query(...)
       if (!result.rows[0]?.is_premium) return res.status(403).json(...)
       next()
     } catch (err) {
       next(err)
     }
   }
   ```
2. Add premium status caching (5 min TTL per userId) to avoid DB hit on every premium route
