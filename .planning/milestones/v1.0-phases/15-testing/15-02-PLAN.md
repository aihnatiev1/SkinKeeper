# Plan 15-02: Flutter Unit Tests & End-to-End Tests

## Goal
Establish comprehensive test coverage for the Flutter app: unit tests for providers/services, widget tests for key components, and E2E integration tests for critical user flows.

## Context
Flutter app with Riverpod, GoRouter, 58+ Dart files. No existing tests. After Phase 14 refactoring, the app will have cleaner widget decomposition, SellService, SelectionNotifier, and family providers — all testable. Key flows to cover: auth, inventory browsing, selling, trades, portfolio.

---

## Tasks

### Task 1: Test Infrastructure Setup
**Files:** `pubspec.yaml`, `test/`, `integration_test/`

1. Add test dependencies:
   ```yaml
   dev_dependencies:
     flutter_test: sdk: flutter
     integration_test: sdk: flutter
     mocktail: ^1.0.0          # Mock generation
     network_image_mock: ^2.0.0 # CachedNetworkImage in tests
     patrol: ^3.0.0            # E2E test runner (or integration_test)
     golden_toolkit: ^0.15.0   # Screenshot testing (optional)
   ```
2. Create test directory structure:
   ```
   test/
   ├── helpers/
   │   ├── test_app.dart        # ProviderScope + MaterialApp wrapper
   │   ├── mocks.dart           # Mock ApiClient, mock providers
   │   └── fixtures.dart        # Sample InventoryItem, TradeOffer, etc.
   ├── core/
   │   ├── api_client_test.dart
   │   ├── cache_service_test.dart
   │   └── theme_test.dart
   ├── features/
   │   ├── inventory/
   │   ├── portfolio/
   │   ├── trades/
   │   ├── auth/
   │   └── settings/
   └── widgets/
       ├── glass_card_test.dart
       ├── premium_gate_test.dart
       └── price_text_test.dart
   integration_test/
   ├── app_test.dart
   ├── inventory_flow_test.dart
   ├── sell_flow_test.dart
   └── trade_flow_test.dart
   ```
3. Create test helpers:
   ```dart
   // test/helpers/test_app.dart
   Widget createTestApp({List<Override> overrides = const []}) {
     return ProviderScope(
       overrides: overrides,
       child: MaterialApp.router(routerConfig: testRouter, theme: AppTheme.dark()),
     );
   }
   ```
4. Create mock fixtures:
   - `sampleInventoryItem()` — with configurable params
   - `sampleTradeOffer()` — pending/accepted/declined variants
   - `samplePortfolioSummary()` — with P/L data
   - `samplePriceData()` — multi-source prices
   - `sampleUser()` — free/premium variants

**Commit:** `test(flutter): test infrastructure, helpers, mocks, fixtures`

### Task 2: Provider Unit Tests
**Files:** `test/features/*/`

1. **SelectionNotifier tests:**
   - `toggle(assetId)` adds/removes correctly
   - `isSelected(assetId)` returns accurate state
   - `selectRange(ids)` batch updates in single notification
   - `clear()` resets state
   - `selectedCount` getter

2. **InventoryNotifier tests:**
   - `build()` fetches from API and returns items
   - `build()` falls back to cache on API failure
   - `refresh()` invalidates cache and refetches
   - Filter/sort providers produce correct output
   - Grouping logic: items with same market_hash_name grouped

3. **PortfolioNotifier tests:**
   - Summary calculation: total value, 24h change, 7d change
   - P/L calculation: profit items, loss items, totals
   - History data point mapping

4. **TradesNotifier tests:**
   - Fetch trades returns paginated list
   - Status filters work (pending, completed, all)
   - Trade detail fetch by ID
   - Sync trigger updates state

5. **AuthNotifier tests:**
   - Login sets user state
   - Logout clears user + invalidates all providers
   - Token expiry triggers logout
   - Session expired triggers session screen redirect

6. **SellService tests:**
   - `quickSell(items)` calculates correct price (market min - 1 kopek)
   - `sellAtPrice(items, price)` sends correct API request
   - Fee calculation accuracy
   - Error handling: session expired, items unavailable

7. **AccountsNotifier tests:**
   - `setActive(accountId)` switches active account
   - `setActive()` invalidates inventory, portfolio, trades providers
   - Premium gate: linking blocked for free users

**Commit:** `test(providers): selection, inventory, portfolio, trades, auth, sell, accounts`

### Task 5: End-to-End Integration Tests
**Files:** `integration_test/`

1. **Auth flow E2E:**
   - App launches → login screen
   - (Mock) login → redirects to portfolio
   - App bar shows user info
   - Logout → returns to login

2. **Inventory browsing E2E:**
   - Navigate to inventory tab
   - Items load and display in grid
   - Search for item by name → filtered results
   - Change sort order → items reorder
   - Change grid columns → layout adjusts
   - Tap item → navigates to detail screen
   - Back button → returns to grid with state preserved

3. **Sell flow E2E:**
   - Select item in inventory
   - Selection tray appears
   - Tap "Sell" → sell bottom sheet opens
   - Set price → confirm → operation starts
   - Progress sheet shows status
   - Quick sell: select → quick sell → confirm → done

4. **Trade flow E2E:**
   - Navigate to trades tab
   - Create trade → partner selection → item selection → send
   - View pending trade
   - Accept/decline trade

5. **Portfolio E2E:**
   - View portfolio summary
   - Switch tabs: overview → P&L → history
   - P/L list shows items with values
   - Add manual transaction → reflected in P&L

6. **Account switching E2E (if multi-account exists):**
   - Go to settings → accounts
   - Switch active account
   - Inventory reloads with new account items
   - Portfolio reflects new account

**Commit:** `test(e2e): auth, inventory, sell, trade, portfolio, account switching flows`
