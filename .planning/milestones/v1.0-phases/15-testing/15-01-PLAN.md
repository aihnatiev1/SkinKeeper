# Plan 15-01: Backend Unit & Integration Tests

## Goal
Establish comprehensive test coverage for the backend: unit tests for all services and utilities, integration tests for API routes with real database, and Steam API mock tests for scraper reliability.

## Context
Backend has 18 services, 11 route files. 5 example test files exist in `services/__tests__/` but no test runner configured. After Phase 14 refactoring, code will have typed errors, SteamClient class, zod validation, TTL caches — all need test coverage.

---

## Tasks

### Task 1: Test Infrastructure Setup
**Files:** `backend/package.json`, `backend/jest.config.ts`, `backend/tsconfig.test.json`

1. Install test dependencies:
   ```bash
   npm install -D jest ts-jest @types/jest supertest @types/supertest
   ```
2. Configure Jest:
   - `ts-jest` transform for TypeScript
   - Test match: `**/__tests__/**/*.test.ts`
   - Setup file: `backend/src/__tests__/setup.ts`
   - Coverage thresholds: statements 70%, branches 60%, functions 70%, lines 70%
3. Create test database setup:
   - Use test-specific `DATABASE_URL` (e.g., `skinkeeper_test`)
   - Run migrations before test suite
   - Truncate tables between test files
   - Teardown: drop test DB after suite
4. Add npm scripts:
   ```json
   "test": "jest",
   "test:watch": "jest --watch",
   "test:coverage": "jest --coverage"
   ```
5. Create test helpers:
   - `createTestUser()` — inserts user + steam_account, returns IDs + JWT
   - `createTestInventory(accountId, items[])` — seed inventory items
   - `createTestPrices(items[])` — seed price_history rows
   - `authenticatedRequest(app, jwt)` — supertest agent with Bearer token

**Commit:** `test(infra): jest + ts-jest + supertest, test DB, helpers`

### Task 2: Utility & Cache Unit Tests
**Files:** `backend/src/services/__tests__/ttlCache.test.ts`, `backend/src/services/__tests__/crypto.test.ts`, `backend/src/middleware/__tests__/validate.test.ts`

1. TTLCache tests:
   - Set and get within TTL
   - Get after TTL expiry returns undefined
   - MaxSize eviction (oldest entry removed)
   - Clear all entries
   - Concurrent access safety

2. Crypto utility tests:
   - Encrypt/decrypt roundtrip
   - Different keys produce different ciphertext
   - Decrypt with wrong key fails gracefully
   - Empty string handling

3. Zod validation middleware tests:
   - Valid input passes through
   - Invalid input returns 400 with error details
   - Missing required fields
   - Type coercion (string "123" → number where applicable)
   - Array size limits enforced

4. Currency/exchange rate tests:
   - Rate conversion accuracy
   - Fallback to 1:1 on fetch failure
   - Cache TTL respected

**Commit:** `test(unit): TTLCache, crypto, validation, currency tests`

### Task 4: Steam Client & Scraper Tests
**Files:** `backend/src/services/__tests__/steamClient.test.ts`, `backend/src/services/__tests__/scrapers.test.ts`

1. **SteamClient tests (with mocked axios):**
   - Successful request returns data
   - 429 → retry with backoff → success on retry
   - 5xx → retry → give up after max retries
   - 302 redirect to login → SessionExpiredError
   - 403 → SessionExpiredError
   - Concurrent request limiting
   - Cookie jar per account isolation
   - Request timing metrics recorded

2. **Scraper tests (with fixture HTML):**
   - Save real Steam HTML responses as fixtures in `__tests__/fixtures/`
   - Trade token extraction: valid page, missing token, changed HTML structure
   - Trade ban date parsing: English format, comma format, "until" format, no date
   - Inventory context 2: normal items, paginated, empty
   - Inventory context 16: trade-banned items, cooldown dates
   - Trade offers HTML: active offers, empty state, mixed sent/received
   - Trade history HTML: completed trades, date parsing, synthetic ID generation

3. **Price crawler tests:**
   - Skinport API response parsing
   - CSFloat API response parsing
   - DMarket API response + HMAC signing
   - Steam Market price parsing
   - Graceful handling of malformed responses

**Commit:** `test(steam): SteamClient retry/rate-limit, HTML scraper fixtures, price crawlers`

### Task 5: API Route Integration Tests
**Files:** `backend/src/routes/__tests__/*.test.ts`

1. **auth.test.ts:**
   - Login flow returns JWT
   - Protected route rejects without token
   - Expired token returns 401
   - Premium route rejects free user with 403

2. **inventory.test.ts:**
   - GET /api/inventory — returns items for active account
   - GET /api/inventory?accountId=X — returns items for specific account
   - Empty inventory returns empty array
   - Pagination (offset, limit)

3. **portfolio.test.ts:**
   - GET /api/portfolio — returns holdings with latest prices
   - GET /api/portfolio/history?days=7 — returns daily snapshots
   - Holdings value calculation accuracy

4. **trades.test.ts:**
   - POST /api/trades — validates input (zod schema)
   - GET /api/trades — returns trades for active account
   - GET /api/trades/:id — returns trade detail
   - Invalid trade ID returns 404

5. **transactions.test.ts:**
   - GET /api/transactions — returns paginated transactions
   - Filter by type (buy/sell)
   - Date range filter
   - Stats endpoint returns correct totals

6. **alerts.test.ts:**
   - CRUD operations (create, read, update, delete)
   - Validation: threshold > 0, valid market_hash_name
   - Premium-only alert types return 403 for free users

7. **market.test.ts:**
   - POST /api/market/sell — validates priceInCents
   - POST /api/market/bulk-sell — validates array + rate limit
   - Missing session returns appropriate error

8. **admin.test.ts:**
   - Requires admin secret
   - GET /api/admin/price-health — returns health status
   - GET /api/admin/cache-stats — returns cache info
   - Rejects without secret (401)

**Commit:** `test(routes): integration tests for all API endpoints`
